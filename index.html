Analyzing C Programs Using LLVM Framework: Implementing Simple Analytics Pass GitHub

This project uses LLVM, a popular compiler framework for a large class of programming languages, that implements a pass that will perform simple analytics on an input program written in C i.e. compute the number of functions, instructions and signed division instructions

Dynamic Analysis and Mutation Strategies: Automated Testing for Divide-by-Zero Errors in C Programs using LLVM Pass and Fuzzing Techniques GitHub

The project uses an automated testing tool leveraging LLVM pass and fuzzing techniques to target divide-by-zero errors in C programs. This project has three distinct parts: firstly, the implementation of a runtime sanitizer through an LLVM pass to perform runtime checks and generate code coverage metrics, secondly, the creation of a mutation-based fuzzer to automatically test programs and provoke potential crashes, and finally, the enhancement of the fuzzer's capabilities to make informed choices based on past test outputs using code coverage metrics. The tool aims to demonstrate the efficacy of random testing while fostering a deeper understanding of dynamic analysis, mutation strategies, and their application in testing software robustness against unpredictable scenarios.

Static Dataflow Analysis Implementation in LLVM Compiler Infrastructure: Chaotic Iteration Strategy and Strategy Pattern GitHub

The project entails the implementation of two intra-procedural dataflow analyses—specifically, Liveness Analysis and Reaching Definitions Analysis—utilizing the Chaotic Iteration algorithm and employing the Strategy Pattern design approach. These types of analyses serve as fundamental building blocks for software development tools. They aid in identifying variables' lifespans (Liveness Analysis) and determining where these variables obtain their values (Reaching Definitions Analysis). In practical terms, these techniques empower modern code linters and Integrated Development Environments (IDEs) to highlight unused variables or potential sources of errors within code, even before execution.
